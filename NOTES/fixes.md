# Fixes

## FIX-001 — Status label layout stomping during long filenames
**Date**: 2026-04-12  
**Problem**: Single status label caused long filenames to push speed/ETA off screen or overlap.  
**Fix**: Split into two dedicated labels — `_filename_lbl` (wrapping) and `_meta_lbl` (single line). Added `wraplength=WRAP` (532px) so filenames wrap naturally without truncation.  
**File**: `app.py:120–135`

## FIX-002 — ffmpeg not found in distributed `.app`
**Date**: 2026-04-12  
**Problem**: Packaged `.app` shipped without ffmpeg, causing merge failures for split video+audio streams.  
**Fix**: Added `static-ffmpeg` to requirements and build pipeline. `_bundled_ffmpeg()` detects PyInstaller context and points yt-dlp at the bundled binary via `ffmpeg_location`.  
**File**: `app.py:55–59`, `build.sh`, `requirements.txt`

## Code Review DEC-100 Fixes (2026-04-12)

## FIX-003 — Thread-safety: StringVar accessed from background thread (D2)
**Problem**: `_worker()` called `self._quality_var.get()` and read `self._save_dir` from the daemon thread. tkinter is not thread-safe; this is a race condition.  
**Fix**: Both values are captured on the main thread inside `_start_download()` before the thread launches, then passed as arguments: `threading.Thread(target=self._worker, args=(url, quality_key, save_dir))`. Worker never touches tkinter state directly.  
**File**: `app.py:191–204, _worker signature`

## FIX-004 — No WM_DELETE_WINDOW handler — thread leak on close (D3)
**Problem**: Closing the window mid-download left the daemon thread running silently; partial files accumulated; Python process could hang on exit.  
**Fix**: `self.protocol("WM_DELETE_WINDOW", self._on_close)` registered in `__init__`. `_on_close()` sets the cancel event and polls via `_wait_and_close()` (100ms intervals, max 3 s) until the worker exits, then calls `self.destroy()`.  
**File**: `app.py:96, 148–164`

## FIX-005 — No URL validation — garbage input reached yt-dlp (D4)
**Problem**: Any string (phone number, file path, plain text) passed directly to yt-dlp, producing raw tracebacks as user-facing error messages.  
**Fix**: `_validate_url()` static method checks for `^https?://` regex before dispatch. `_friendly_error()` maps known yt-dlp error substrings to plain-English copy. `_ERROR_MAP` covers 12 common failure patterns.  
**File**: `app.py:40–50, 65–73, 175–181`

## FIX-006 — Playlist URLs silently downloaded all videos (D5)
**Problem**: Pasting a YouTube playlist URL downloaded every video in the playlist without warning, potentially filling the user's disk.  
**Fix**: `_probe_playlist()` runs `extract_flat` before download to detect playlist type. If count > 1, `_ask_playlist()` shows a native confirmation dialog (blocks worker thread via `threading.Event`, max 120 s timeout). User must confirm before download proceeds.  
**File**: `app.py:212–248`

## FIX-007 — Unpinned dependencies (D6)
**Problem**: `requirements.txt` had bare package names. A yt-dlp update could silently break all downloads.  
**Fix**: Pinned to tested versions: `yt-dlp==2026.3.17`, `customtkinter==5.2.2`, `static-ffmpeg==3.0`.  
**File**: `requirements.txt`

## FIX-008 — Hardcoded absolute path committed in .spec (D7)
**Problem**: `SaveThisVideo.spec` contained `/Users/goralski/...` path, making it non-portable and misleading as a committed artifact.  
**Fix**: Added `*.spec` to `.gitignore`. The spec is regenerated by `build.sh` on every run; there is no value in committing it. Created `.gitignore` covering `.venv/`, `build/`, `dist/`, `*.spec`, `__pycache__/`, `.DS_Store`.  
**File**: `.gitignore` (new)

## FIX-009 — Silent exception in _paste() (D8)
**Problem**: `except Exception: pass` swallowed clipboard errors with no user feedback.  
**Fix**: Replaced with targeted handler that shows `"Nothing on clipboard to paste."` in the meta label. Empty clipboard after stripping is also handled with the same message.  
**File**: `app.py:139–148`

## FIX-010 — Module-level side effects outside __main__ guard (D9)
**Problem**: `ctk.set_appearance_mode("system")` and `ctk.set_default_color_theme("blue")` were called at module level, initialising the Tk subsystem on any import.  
**Fix**: Moved both calls inside the `if __name__ == "__main__"` block alongside `_setup_logging()` and `App().mainloop()`.  
**File**: `app.py:306–309`

## FIX-011 — No version constant (D10)
**Problem**: No `__version__` or version string anywhere. Window title hardcoded. `.spec` had no bundle version.  
**Fix**: `APP_VERSION = "1.0.0"` defined at top of module. Window title set to `f"SaveThisVideo {APP_VERSION}"`. `build.sh` passes `--app-version` (via PyInstaller) when the notarization pipeline is active.  
**File**: `app.py:16, 89`

## FIX-012 — Raw yt-dlp error messages shown to users (S1)
**Fix**: `_friendly_error()` maps 12 known yt-dlp error substrings to plain-English copy. Unmatched errors have the `[extractor] id:` prefix stripped via regex before display.  
**File**: `app.py:65–73`

## FIX-013 — _on_done showed directory but not filename (S2)
**Fix**: `_hook` captures `d.get("filename", "")` when `status == "finished"` into `self._last_saved_path`. `_on_done()` reads it via `os.path.basename()` and shows it in `_filename_lbl`: filename on line 1, save directory on line 2.  
**File**: `app.py:275–287, 294–302`

**Fix**: `_set_status(filename="", meta="Starting…")` in `_start_download()` now explicitly clears `_filename_var` before launching the worker.  
**File**: `app.py:200`

## FIX-015 — tkinter.filedialog import inside method (S4)

## FIX-016 — WRAP magic number with unexplained approximation (S5)
**Fix**: Replaced `WRAP = 532` with named constants: `WINDOW_W = 580`, `SIDE_PAD = 20`, `CTK_LABEL_PAD = 8`, `WRAP = WINDOW_W - SIDE_PAD * 2 - CTK_LABEL_PAD`. The math is explicit and self-documenting.  
**File**: `app.py:21–24`

## FIX-017 — No logging for production diagnostics (S6)
**Fix**: `_setup_logging()` attaches a `RotatingFileHandler` to `~/Library/Logs/SaveThisVideo/app.log` (2 MB max, 3 backups). Module-level `log = logging.getLogger("savethisvideo")` is used throughout. Logs: app start, download start/complete/cancelled/failed, app close. Does not log full URLs (privacy).  
**File**: `app.py:77–95, 98`

## FIX-018 — Pre-flight disk space check missing (S8)
**Fix**: `shutil.disk_usage(self._save_dir).free` checked before download starts. If < `DISK_WARN_BYTES` (500 MB), user sees friendly inline error with free space shown in MB. `OSError` (unmounted/inaccessible path) is silently swallowed — yt-dlp will surface it if it matters.  
**File**: `app.py:183–192`

## FIX-019 — Progress bar stayed at previous state between downloads (S9)
**Fix**: `_reset()` now calls `self._progress.set(0)` unconditionally, so the bar returns to zero on every terminal state (done, cancelled, error). `_on_done()` then sets it to 1.0 after reset to show completion.  
**File**: `app.py:303–309`

## FIX-020 — _save_dir stored as str, initialized from Path (S10)
**Fix**: `self._save_dir` is now typed and stored as `Path` throughout. `_trunc_path()` accepts `Path`. `_browse()` assigns `Path(path)`. yt-dlp `outtmpl` uses `str(save_dir / "%(title)s.%(ext)s")`. `_on_done` and `_start_download` use `self._save_dir` directly.  
**File**: `app.py:91, 152, 253`
